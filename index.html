交互物 ID (Target) 锁定特定物品 (ID) 锁定品质 (Quality) 单项数量 (Qty) 操作

📂 基础数据配置

  • 物品库：每行格式 ID,品质。品质必须匹配左侧设定的四个等级。
  • 随机配额：除了锁定项，系统还能额外抽取的物品总数上限。

🎯 核心分配逻辑

  • 多重掉落：为同一个 Target ID 添加多行，即可实现一个箱子掉落多件物品。
  • 数量(Qty)：指该格物品堆叠数。金币填 500，装备填 1。

🔒 锁定与随机规则

  • 特定ID优先：若填写“锁定ID”，则该槽位固定掉落此物，无视品质设定。
  • 品质随机：若选定品质，则从库中对应品质池内随机抽一，并扣除左侧配额。
  • 完全随机：若两项均空，则消耗剩余配额进行全随机填充。

📋 导出与使用

  • 点击 Generate 生成最终分配名单。
  • JSON 结构包含 target, item_id, quantity 等关键字段，直接对接游戏逻辑。
